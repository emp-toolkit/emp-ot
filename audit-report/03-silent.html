emp-ot — silent OT: cGGM, MPCOT, LPN, Ferret

The low-communication RCOT stack: the Half-Tree correlated-GGM (cGGM) puncturable-PRF tree, the MultiPointGadget that turns a batch of trees into a multi-point COT, the primal-LPN linear-code amplifier and its security-class parameter sets, and the two consumers — Ferret (streaming rollover) and SilentFerret (begin-time prepay, wire-free consume incl. next_chunks_parallel). Malicious integrity rests entirely on a per-round Fiat-Shamir chi-fold consistency check with a fail-closed error() abort.

1 · Role in the stack & the silent-OT loop

Ferret is emp-ot's low-communication RCOT extension: it produces F₂ Δ-correlated random COTs (LSB choice-encoding, the convention shared with IKNP/SoftSpoken) by iterating a bootstrap → MPCOT → primal-LPN loop. Each round consumes M base COTs, builds t punctured cGGM trees of 2^tree_depth leaves, LPN-folds them (d=10 XORs/output) into a large batch of pseudorandom COTs, and spends the round's last refill_trees trees regenerating the next round's M bases — self-bootstrapping.

S0base OTCSW / BMM malicious-secure base (Report 1). Bootstraps the first round's M COTs; refused unless is_malicious_secure() in malicious mode
S1cggm.hHalf-Tree correlated GGM tree: build_sender / eval_receiver, split (bit-reversed) leaf layout, per-level Δ correlation
S2mp_gadget.hMultiPointGadget{Sender,Receiver}<AuthValue> — batch of trees → multi-point COT/VOLE; hosts the chi-fold check
S3lpn.hLpn<AuthValue, d=10> primal-LPN amplifier: d sparse XORs/adds into a size-k public table
S4ferret / silent_ferretthe two consumers: streaming rollover (Ferret) and begin-time prepay + wire-free consume (SilentFerret)

The same three primitives also carry sVOLE over F2ᵏ / Fp (Report 4): the gadget is generic over an AuthValue carrier that supplies the field ops, chi-fold, and wire-format traits. This report covers the Ferret AuthValueFerret shape (F₂, packed check, no per-tree secret sum) and the shared primitive layer; the typed sVOLE carriers appear where they change the contract.

Where malicious security lives

The whole loop's integrity reduces to one per-round check: a random linear combination of the multi-point outputs (the "chi-fold"), seeded from the running Fiat-Shamir transcript digest and verified via a random-oracle commitment (Ferret App. C). Everything else — cGGM correctness, LPN folding, refill — is covered by that check because the refill trees share the round's accumulator (§7). Semi-honest mode elides it entirely (§4).

2 · cGGM — Half-Tree correlated GGM tree

emp-ot/common/cggm.h

The header banner (cggm.h:8-14) documents the Half-Tree construction: at every non-leaf left = H(parent), right = parent ⊕ left, root children (k, Δ⊕k), so the leveled correlation Δ = XOR of all nodes on level i holds for every i ∈ [1,d]. That correlation is what the sender's per-level corrections exploit; H is emp-tool's CCRH.

Split (bit-reversed) leaf layout

Leaves are stored in a bit-reversed "split" layout (cggm.h:25-38): each level writes all left children into the lower half in place and all right children into the upper half, so path p's leaf lands at bit_reverse(p, d). expand_level (cggm.h:66-111) documents the tile invariant that makes the in-place left-write safe — CCRH::H reads the parent tile up front, left writes only touch already-consumed inputs, right writes land strictly above unread parents.

build_sender:  level 1 → leaves[0]=k, leaves[1]=Δ⊕k, K0[0]=leaves[0]
               levels 2..d-1 never clear LSBs (intermediate nodes feed the next AES level)
               level d optionally clears LSBs (ClearLeafLSB)
               emits only the left-side sum K^0_i into K0[i-1];  K^1_i = K^0_i ⊕ Δ is sender-derivable
eval_receiver: zero all Q=2^d leaves, walk levels tracking pos = split-index of the on-path node,
               place alpha_bar-side child from K_recv, recover sibling = sum_pre ⊕ junk ⊕ K_recv[i-1]
               → on return leaves[bit_reverse(alpha,d)] = zero_block (the puncture); all else correct

The ClearLSB/ClearLeafLSB path threads the COT LSB-choice convention into the tree write via kCggmLsbClearMask = makeBlock(0xFF..FF, 0xFF..FE) (cggm.h:63-64), applied to both the written leaf and the accumulated sum so K0 stays consistent on both sides. Both parties must set the same ClearLeafLSB or K0/K_recv desync (cggm.h:172-175). The Tile template parameter is LOCAL-only — it changes batching of CCRH::H, not the computed leaves or wire bytes (cggm.h:42-54).

Receiver privacy = never learning Δ (stated, unenforced)

cGGM receiver privacy rests on the receiver never learning Δ, which makes the punctured leaf leaves[α] pseudorandom (cggm.h:16-20). eval_receiver takes no Δ and only the sender holds cggm_delta (mp_gadget.h:103-105) — but this is a documented invariant with no runtime check: any path that leaked Δ to the receiver would let it reconstruct the hole via the same LSB-clear closure the α-fill uses. Structurally maintained, unverified in code.

d=0 is a latent precondition — but the true floor is d=2, not d=10

build_sender writes leaves[0]/leaves[1]/K0[0] unconditionally before any depth guard (cggm.h:146-163), so d=0 would produce two leaves for a nominally one-leaf tree. It is never triggered — but not because "tree_depth ≥ 10": that bound holds only for Ferret's MPCOT (ferret_b10..b13). SoftSpoken calls cggm::build_sender directly with k ∈ {2,4,8} (softspoken.cpp:209), so the real cross-call floor is d=2 — still never 0, just not for the stated reason.

3 · MultiPointGadget — roles & the AuthValue contract

emp-ot/common/mp_gadget.h

One template class MultiPointGadget{Sender,Receiver}<AuthValue> subsumes three shapes (mp_gadget.h:12-44): Ferret MPCOT (AuthValueFerret, no secret sum, F2kPacked check, kClearLeafLSB=true), and Mpsvole over F2ᵏ / Fp (val+mac pair, per-tree secret_sum on the wire, FTyped check, kClearLeafLSB=false). The carrier supplies field ops, auth_from_block, expand_chi, accumulate_VW, and the wire-format traits; a wrong run_end variant trips a static_assert (mp_gadget.h:272-273,335-336).

Per-tree flow

  • Sender run_next_tree (mp_gadget.h:148-183): draw a fresh cGGM root seed from prg, build the tree, ship c[j] = base_i[j] ⊕ K0[j] over tree_depth blocks; if kHasSecretSum also ship secret_sum; flush; then in malicious mode snapshot chi_seed = io->get_digest(), expand chi, fold VW.
  • Receiver run_next_tree (mp_gadget.h:406-466): derive α as the MSB-first NOT-of-LSB of the base COTs (α_j = !getLSB(base_i[j])), receive c[], set K_recv = base ⊕ c, run eval_receiver, α-fill the punctured slot. The F2kPacked α-fill is leaves[rev] = XOR(all leaves) ⊕ bit0_mask — the LSB-clear closure recovers the sender's cleared α-leaf and re-injects the carrier bit (mp_gadget.h:425-437); the FTyped α-fill instead solves leaves[rev].mac = triple_yz − (secret_sum + Σ_{j≠α} mac).
Verified — Ferret's low-communication property

AuthValueFerret sets kHasSecretSum=false (ferret.h:49): the receiver never receives a per-tree secret sum (the recv_data is guarded by if constexpr (kHasSecretSum), mp_gadget.h:420). The only per-tree send is the tree_depth-block correction c[], and the hole is reconstructed locally as XOR(all LSB-cleared leaves) ⊕ bit0_mask — this is the source of Ferret's small communication.

The α = getLSB(base) precondition is unchecked

The gadget reads α purely from getLSB(base_i[j]) (mp_gadget.h:408-411,546-549) and the sender's correction is base ⊕ K0, so correctness depends on the outer extension delivering base COTs whose bit-0 LSB equals the receiver's choice bit. Ferret satisfies this by design (the RandomCOT contract, ot.h:128-131), but the gadget itself performs zero runtime verification of the precondition.

Decoupled prepare_all / produce_tree (SilentFerret path)

A second path (mp_gadget.h:185-267,468-537) expands/evaluates all trees in one batched flush, remembering per-tree cGGM root seeds (sender, reproducible) or received corrections (receiver, retained by caller). The comment claims the socket sends and per-tree get_digest() snapshots stay on the owning thread in tree order, so the malicious transcript is bit-identical to the interleaved path; only compute (cGGM + VW) moves to the pool. That equivalence is a correctness-critical claim — see §8/§9.

4 · Malicious chi-fold check — flavors & abort

emp-ot/common/mp_gadget.h

Chi seeds are pulled from the running FS transcript via io->get_digest() after the tree's corrections were sent/received (mp_gadget.h:177-182,458), so both parties derive identical chi with no extra wire traffic and the check binds to the sent messages — get_digest = H(d_AB ‖ d_BA) over the FS transcripts that send_data/recv_data already fed the corrections into (io_channel.h:157-168). This "chi after corrections" ordering is exactly what makes App. C sound.

Abort authority is receiver-only

Every error()-based abort is on the no-Δ receiver side; the sender only squeeze_digests and send_datas its digest and never receives or compares one (mp_gadget.h:286-288,325-326,345). This matches the paper's one-sided check: the receiver reveals its chi-fold opening (x_prime/x_star) unconditionally before learning the sender's response, and the argument that this leaks nothing about α is inherited from Ferret, not re-derived here.

The check binds to fresh session secret, not just sid

The RO domain-separates on kDomCheckPacked/kDomCheckTyped plus the per-session sid — but it also absorbs the session-fresh secret consistency value r1/vb (mp_gadget.h:283-286,341-344), where r1 folds this session's VW and packed COTs. So reusing a sid across sessions does not make the commitment replayable — each session's r1 differs and the verifier recomputes and equality-checks it. sid provides domain separation only; nothing enforces sid uniqueness (default zero_block, ot.h:29-32), and that is harmless for the check's soundness. Ferret still sets gadget.sid = sid.value() before run_begin (ferret.cpp:176-180).

Semi-honest mode ships and checks nothing — by design

Every check method early-returns on !is_malicious (mp_gadget.h:275,338,566,646). In semi-honest mode a malicious sender can hand inconsistent cGGM trees and go undetected; the only detector — the receiver's error() — sits inside the guarded methods. Soundness in that mode is only against a semi-honest adversary. A static_assert (mp_gadget.h:51-52) pins Hash::DIGEST_SIZE == 2·sizeof(block) so the packed 2-block digest buffer cannot silently overflow.

5 · Primal-LPN amplifier

emp-ot/common/lpn.h

Lpn<AuthValue, d=10> (lpn.h:34-112) folds d=10 pseudo-random indices idx = r & (k-1) into pre[idx] per output via AuthValue::auth_add_into — the implicit d-regular sparse code, n outputs each a XOR/sum of d columns of a k-wide table. Randomness is a sequential PRG keyed by reseed(seed); Ferret derives that seed as RO("emp-ot:ferret:lpn-seed", sid).squeeze_block() locally on both parties with no wire exchange (ferret.cpp:29-31,182-185) — the LPN matrix is public, so this is safe by design.

k must be a power of two (caller-enforced only)

idx = r & (k-1) is uniform on [0,k) with no rejection only because k is a power of two; the ctor's mask(k_-1) has no assert (lpn.h:20-22,72). The sole enforcement is PrimalLPNParameter deriving k = 1<<logk (tuning.h:37) — purely structural.

Reduction cadence & parallel slicing

For non-F₂ fields the fold does kStep = min(kLpnSafeAddsPerReduce, d) adds between partial reduces, then a final reduce per output (lpn.h:44-68). For F₂ (Ferret, F2k) kLpnSafeAddsPerReduce = INT_MAX (XOR has no carry → all d adds in one batch); for Mersenne Fp it is 5 (fp_vole.h:78). blocks_per_chunk(length) (lpn.h:84-92) gives the exact PRG-block consumption of a chunk deterministically in (length, d), so a seek'd, caller-owned PRG lets compute_slice be sliced across threads order-independently (lpn.h:98-105). The M=32 batched / 4-batched / scalar-tail split is pure scheduling — all three produce identical outputs, so parties may differ in batch width freely.

Fp reduce cadence vs d=10 — unchecked, untested

fp_vole.h:74-78 notes mod-p adds overflow uint64_t after ~8 unreduced accumulations, hence the 5+5 split at kLpnSafeAddsPerReduce=5. No static_assert ties 5 to d=10, and test_fp_vole.cpp references neither the constant nor overflow — a too-large value would silently overflow the accumulator before reduce. It is a correctness invariant tied to the carrier type, exercised at exactly one field size.

Type-pun smell in compute_block (bounded, not a live bug)

compute_block type-puns the PRG buffer from block[] to uint32_t* via a C-style cast (lpn.h:50), leaning on __m128i's may_alias in one direction only — a strict-aliasing smell. The exact-count bookkeeping (lpn.h:47 vs lpn.h:84-92) is consistent. The index cap is k ≤ 2³⁰ (not 2³¹ — mask and k_ are int, and 2³¹ overflows a signed int), well clear of the deployed logk=19 (tuning.h:109).

PrimalLPNParameter (tuning.h:27-40) is a constexpr bundle: given (t, logk, tree_depth) it precomputes k = 1<<logk, M = k + t·tree_depth + 128, and refill_trees = ceil(M / 2^tree_depth). The +128 is kConsistCheckCotNum folded in (tuning.h:20-22).

The user-visible budget per round is t − refill_trees = 1832 / 1451 / 1099 / 777 trees respectively — the round spends its last refill_trees trees seeding the next round's M bases (ferret.cpp:98).

The 128-bit claim is an external, non-reproducible assertion

tuning.h:103-108 asserts each set gives ~128-bit security against published regular-LPN attacks (Gauss / SD-ISD / BJMM / hybrid / esser / agb2) "per the lpnestimator," with the hybrid attack binding and each t the largest the (logk, tree_depth) pair admits at the 128-bit floor. No estimator, script, or attack-cost table is vendored anywhere in the repo — a whole-tree scan for lpnestimator/SD-ISD/BJMM and a git ls-files for .py/.sage/.sh turn up only this comment. The security claim on t/logk/tree_depth is a constexpr-encoded external result that cannot be reproduced from the codebase alone. These are SECURITY-class parameters (docs/performance-tuning.md:20-25,121: "do not touch").

A numeric coupling between two files

ferret_b10.M ≈ 74,164 drives two things: it is the tiered-bootstrap inner param (ferret.cpp:156-167), and softspoken_ferret_bootstrap_chunk_blocks = 580 (= 74,240 OTs/chunk) is sized to fit b10.M in one SoftSpoken chunk (tuning.h:88-92). Changing b10 without re-deriving 580 would silently over- or under-produce in the bootstrap chunk — a cross-file coupling with no compile-time link.

7 · Ferret — lifecycle, bootstrap, wire format

emp-ot/ot_extension/ferret/ferret.{h,cpp}

Ferret (ferret.h:107) is an OTExtension → StreamingExtension<block> RCOT backend; default param is ferret_b13, default base OT is CSW (ferret.h:20,114). Two ping-pong buffers carry_curr_/carry_next_ carry the round; slots [0,M) hold the base COTs, laid out as [0,128) check COTs, [128,128+k) LPN-secret pre-COTs, and [128+k, M) cGGM-correction COTs (tree i at 128+k+i·tree_depth) (ferret.cpp:193-196, silent_ferret.cpp:254).

begin() : enter_session → bootstrap_ (lazy, once) → swap curr/next → tree_idx=0 → inner_run_begin_
next()  : if tree_idx == t-refill_trees  → end(); begin()   (transparent auto-rollover)
          process_one_tree_: build cGGM · ship c[] · (mali) VW-accumulate ; LPN compute_slice ; tree_idx++
end()   : run_refill_ (last refill_trees trees → carry_next_) → inner_run_end_ (chi-fold check) → exit_session_
Verified — the refill trees are covered by the same check

run_refill_ writes the refill trees into carry_next_, which begin()'s swap makes the next round's base COTs (ferret.cpp:219-228,87). Both user trees and refill trees call the same gadget->run_next_tree, accumulating into the shared consist_check_VW[tree_idx] array sized tree_n = param.t; run_end_packed folds over all slots. So a cheating sender cannot corrupt the material that bootstraps the next round without failing this round's chi-fold check.

Bootstrap & base-OT trust enforcement

bootstrap_ (ferret.cpp:131-169) is idempotent and enables the FS transcript before either path. Tiered: if param.M > 2·b10.M a nested Ferret(b10) supplies the M bases (b10 itself drops to SoftSpoken since its M≈74k is below threshold — at most one nesting level); otherwise SoftSpoken<8> with a 580-block chunk directly. The receiver's choice randomness flows end-to-end: the top-level receiver forwards a sub-seed from its choice_prg to the inner source's set_choice_seed (ferret.cpp:148-152), so the inner base-COT LSBs become the MPCOT α positions.

Verified — malicious mode refuses a non-malicious base OT

The shared OTExtension ctor runs if (malicious_ && !base_ot->is_malicious_secure()) error("…requires a malicious-secure base OT") (ot_extension.h:152-153) — a runtime abort at construction, not an NDEBUG-elidable assert. Ferret defaults malicious=true with a CSW base (is_malicious_secure()==true, csw.h:51); a plain OT base (false, ot.h:16) hits the abort. The nested b10 / SoftSpoken re-run the same check on the moved base.

bootstrap_ leaves csw_base_ dangling

The OTExtension ctor caches csw_base_ = static_cast<CSW*>(base_ot.get()) (ot_extension.h:156). bootstrap_ std::moves base_ot into a nested b10/SoftSpoken instance local to the function, destroyed when it returns (ferret.cpp:156-157) — leaving csw_base_ aliasing a freed/moved-out CSW for the rest of the Ferret's life. It is never dereferenced today (Ferret uses the nested instance's deferred check), but set_sid's !setup_done guard is assert-only — a release-build post-bootstrap set_sid dereferences the now-null base_ot (ot_extension.h:102,156). A latent use-after-free / null-deref keyed on csw_base_ != nullptr.

LPN arity is hardcoded, not read from tuning::lpn_d

ferret.cpp:41 and ferret.h:174 both pin the literal Lpn<AuthValueFerret, 10>. tuning::lpn_d = 10 (tuning.h:116) is only reachable via lpn.h:34's default template arg, and no Ferret/svole call site omits the third arg — so retuning tuning::lpn_d alone is a no-op for Ferret and would desync it from any code that does read the constant. They coincide at 10 today; low severity, sharp edge.

8 · SilentFerret — prepay & wire-free consume

emp-ot/ot_extension/ferret/silent_ferret.{h,cpp}

SilentFerret subclasses Ferret and moves all wire traffic into begin(): begin_batch_ (silent_ferret.cpp:57-148) prepays K = ceil(n_ots / cots_per_round()) rounds — shipping every round's corrections and running one batched malicious check up front — then repositions the consumer at round 0 so next()/produce_range()/next_chunks_parallel() emit COTs with zero I/O. A global tree counter G = abs_round·t + local_tree keys both re-derivation streams: the LPN fold at G·bpc_ and the seed PRG seeked to G (silent_ferret.cpp:261-270, silent_ferret.h:120-127), so it is order-independent — produce_range from disjoint threads is bit-identical to serial next().

  • Sender re-derivation: cggm_seed_key_ = gadget_send_->prg.seed() captured once (silent_ferret.cpp:78-81); tree g's root seed = PRG(cggm_seed_key_).seek(abs_round·t + local). No corrections stored.
  • Receiver re-derivation: cannot re-derive corrections, so it stores them in c_rounds_ (K·t·tree_depth blocks, silent_ferret.cpp:94-95,121-124); produce_tree re-evaluates from base_i + stored c_i with no I/O.
  • Batched check: acc_vw_/acc_phi_ accumulate each round's XOR-folded VW/phi with no I/O (mp_gadget.h:301-306,600-607); one finalize round-trip masked by round-0's pristine 128 check COTs base0_, snapshotted before any check mutates the region and restored afterward (silent_ferret.cpp:91-92,137-141,145).
Batched multi-round check soundness rests on FS append-only ordering

The batched check folds K rounds' VW into one F(2¹²⁸) accumulator and runs a single round-trip; its soundness reduces to Ferret App. C over m = K·t only if the FS transcript stayed append-only in (round,tree) order across the prepay loop (silent_ferret.cpp:133-136, mp_gadget.h:314,613). This holds because one global Δ (ferret.cpp:49,77) lets a single mask cover all rounds and each round's chi is sampled post-commitment via get_digest() on the owner thread. High-severity design dependency: the reduction is only as good as that ordering discipline.

Counter reuse across begin() repeats chi/LPN streams

end() is the only writer of abs_round_ (advancing it by n_rounds_, silent_ferret.cpp:156); begin() reads it into abs_round_base_ (silent_ferret.cpp:87). Since both the LPN and cGGM-seed PRGs fork from G = abs_round·t + local, a begin() reusing a non-advanced abs_round_ repeats those pseudorandom streams. A skipped-end()/double-begin is caught by enter_session_'s tripwire assert (streaming_extension.h:149) — so reuse is silently reachable only under NDEBUG. end() also rolls any unconsumed prepaid rounds forward to base_K so a following begin() continues the base chain without counter reuse (silent_ferret.cpp:150-158).

The prepare_all bit-identity invariant is unchecked

The claim that the decoupled prepare_all/produce_tree path yields a malicious transcript bit-identical to serial run_next_tree rests entirely on the discipline that all socket sends and per-tree get_digest() FS snapshots stay on the owning thread in tree order while only pure compute is pooled (mp_gadget.h:196-198,235-240,499-506). No lambda touches io; cggm.h has no I/O; but it is structurally maintained, not runtime-asserted, and no pooled test verifies transcript identity (trace_hash runs single-threaded; the pooled check only proves mutual consistency, not equality to the serial path).

gadget_send_->prg must never advance after key capture

Sender correctness depends on gadget_send_->prg never advancing after cggm_seed_key_ is captured — it must use prepare_all/produce_tree, never run_next_tree (silent_ferret.cpp:71-81). SilentFerret overrides process_one_tree_ to use produce_trees_, and prepare_all/produce_tree take explicit seed params rather than drawing from the PRG. If any path called Ferret::run_next_tree on the sender gadget, the re-derived seeds would diverge from the prepaid ones and produce silently wrong COTs. A load-bearing invariant with no runtime guard.

9 · Threading & next_chunks_parallel

emp-ot/ot_extension/ferret/silent_ferret.cpp emp-ot/common/mp_gadget.h

next_chunks_parallel (silent_ferret.cpp:166-209) is a cursor-ordered bulk draw with next()'s auto-rollover: each current-round slice is split across up to T short-lived worker threads calling the const, disjoint-range produce_range; the cursor (local_tree_) advances only on the main thread after join(). Begin-time expansion is similarly parallel — prepare_all pools the cGGM expand + VW inner-product a batch at a time (batch_ = n_threads·8) while keeping every send and get_digest() on the owning thread.

Verified — parallelism is soundly structured against races

prepare_all keeps all socket I/O and FS snapshots serial in tree order and farms out only pure compute over disjoint per-task scratch (s·leave_n / s·tree_depth strides); produce_trees_/produce_tree are const with caller-supplied kr/k0 scratch and thread-forked PRGs seeked to G0·bpc_/G0; next_chunks_parallel joins each wave before mutating cursor state (silent_ferret.cpp:186-208, mp_gadget.h:222-253,499-524). The only concurrency artifact is benign begin-time false sharing on adjacent consist_check_VW[i] slots.

Mixing produce_range with the cursor duplicates COTs

produce_range is const and does not advance local_tree_; the cursor path (next/next_n/next_chunks_parallel) does. Because produce_trees_ is deterministic in (abs_round, local), calling produce_range then next() in the same prepaid round re-emits the same trees = duplicate COTs. The hazard is documented (silent_ferret.h:104-109) but unenforced at runtime — the only nearby assert (silent_ferret.cpp:235-237) is a range-bounds check, compiled out under NDEBUG, and does not even detect in-bounds mixing.

Auto-rollover past the prepaid budget does live I/O inside a "wire-free" draw

ensure_tree_available_ (silent_ferret.cpp:211-223), reached from both next() and next_chunks_parallel, rolls wire-free into the next prepaid round if one exists — else it performs a full live end(); begin(batch_n_ots_) with blocking correction exchange, inside a draw the header markets as wire-free. If the two peers' cumulative draw counts diverge (one draws past prepared_capacity(), the other does not), one side blocks in the reprepay's round-trip while the other never enters it — a silent distributed hang, not an error, with no runtime cross-check of draw totals.

Streaming semantics come from StreamingExtension: run()/next_n() drain a leftover buffer so non-chunk-multiple draws don't pay a fresh chunk; enter_session_ resets leftover on begin() so no stale tail crosses a session; a destruction in-session calls error() always-on (streaming_extension.h:133-139,152-153). kMaxTreeDepth=32 caps the stack scratch in produce_trees_, asserted against param.tree_depth in the ctor (production max is 13).

The reverted LPN tuning (7efb923 ← 0b61590)

Commit 7efb923 fully reverts the per-platform lpn_batch_m 32→64 + size-gated gather-prefetch change: current lpn.h has no prefetch code and tuning.h has a single unconditional lpn_batch_m = 32 (tuning.h:120). The documented reason (docs/performance-tuning.md:26-75): the same change was +9.8% on Zen 5 but a regression on Apple M4 and −4.1% on Granite Rapids at the production 8 MiB table, and no queryable machine quantity (cache size) separates the Zen5/Granite behaviors — so neither a fixed constant nor a runtime formula is safe, and defaults must be safe on un-swept hardware. The Zen-5 recipe is parked for a future "make tune."

Commit 37e2576 drops the local lsb_only_mask = makeBlock(0LL,1LL) in favor of emp-tool's bit0_mask = makeBlock(0,1) (block.hpp:57) — byte-identical, and the exact bitwise complement of kCggmLsbClearMask (…FE). A pure rename/de-dup with no functional change. Any future autotuner is required to enumerate only a registry of LOCAL knobs and emit tuning_local.h with provenance; a value inside the noise band is not emitted — a plan, not yet implemented code.
